# 問題点と改善提案

## 実施済みの改善
- モジュール分割を実施: `app / camera / preferences / inference / pipeline / ui` の責務を分離。
- 設定永続化を実装: `ConfigStore` でカメラ選択とパイプライン設定を TOML に保存。
- 推論障害時フォールバックを実装: `HandTrackingProcessor` 初期化失敗時に `NoopProcessor` へ切替。
- 推論安定化を実装: ROI追跡、ランドマーク平滑化、検出妥当性チェックを導入。

## 残課題（次の改善候補）

1) ジェスチャー入力の追加
  - クリック、ドラッグ、スクロールなどの操作判定をランドマーク時系列から実装する。

2) 推論スループットの最適化
  - 推論間隔・ROIサイズ・補間係数を環境別にチューニングし、FPSと操作遅延を両立する。

3) クロスプラットフォーム強化
  - Windows以外でもカーソル制御が有効になるようOS別バックエンドを実装する。

4) 自動テストの強化
  - 座標変換、ROI更新、検出妥当性判定、設定ロード/セーブのユニットテストを整備する。
